Explain Control Flow And Loops Using An Example Process From Everyday Life.

What Is Control Flow In Javascript?

Control flow describes the way our computer performs tasks and the order that it completes them in. The control flow runs top to bottom through our Javascript code, checking that all the actions have been completed. In the occurrence that a specific action has not been completed, the control flow will halt and activate another set of actions which notify the end-user or developer.

This would be akin to following a prewritten shopping list while doing the groceries. Although we have already specified all the items which we must find and purchase before leaving the supermarket, it is common that we would over-look a specific written item and have to go back down an aisle we have already passed. Having our shopping list reminds us of all the ‘actions’ we must complete before leaving the supermarket.

What Are Loops in Javascript?

Creating a loop in Javascript enables us to repeat an action many times over. Using loops shortens the amount of code we would have to type manually, and in doing so, saves us valuable time.

This would be akin to using a power drill over a screw driver when building a house. Although a screw driver is fully capable of completing the work needed to build our house, we would save a lot of time and energy using the power drill.

Describe What The DOM is And An Example Of How You Might Interact With It.

What is the DOM?

The DOM or Document Object Model is the “middleman” between our webpages HTML document and our webpages Javascript functionality. More specifically the DOM is a programming interface that represents our HTML ‘Document’ and its contents as ‘Objects’ and nodes. Furthermore, the ‘Model’ within the DOM represents the hierarchy of these objects as a “tree-like structure", in relation to the layout of our HTML elements. This representation of our HTML document as objects/ nodes and how these objects/nodes are laid out, both enables communication with Javascript and the ability to use Javascript to manipulate our HTML.

A simple way we can interact with the DOM, is through the use of <script></script> tags within the body of our HTML document. Using these <script> tags enables us to create Javascript code within our HTML. We can then begin to write our Javascript code and manipulate our document through the use of Javascript functions, variables and loops.

An Example of Accessing the DOM With <script> Tags.

<script>
  alert("Welcome to my Technical Blog!");
</script>

In the above example, we have interacted with our DOM by creating a Javascript function within the body of our HTML document. Explain The Difference Between Accessing Data from Arrays and Objects.

How do we access data from an Array?

In Javascript, an array is a list-like object which stores ‘data’ as a single variable. The data within an array, follow the arrays given name and is stored within square brackets separated by commas. To access an arrays data we must create a separate line of code and ‘call’ the array's name followed by a number within square brackets. This number specifies the desired data we wish to call and the order in shows within the array. Although we can include text (strings) as array data, it is still considered a number when discussing the order which it appears. An array stores data through zero-based indexing so when we call the arrays ‘first’ data entry, we must start from the number 0 and so forth.

Example of an Array.

var fruits = ['Banana’, ‘Apple, ‘Orange’]

Above is a small example of an array. We have specified the name of the array as “fruits” and within it contains three relating data entries.

var first = fruits[0] // Banana

Above is an example of how we would access data from our specified array, using both the arrays name and the order in which our desired data appears in. Following the two backslashes is the resulting output from our call.

How do we access data from an Object?

In Javascript, an Object (much like an array) is used to store data pertaining to our code. The data within our object is displayed indented in property name/value pairs and is wrapped in curly brackets following the objects given name. We can access a objects data through two means: one being (without spaces) the objects name followed by a period, followed by the properties name, while the second option (without spaces) is the objects name followed by a square bracket, followed by double quotes, followed by the properties name and finished with closing double quotes and closing square bracket.

Example of a an Object.

var fruits = {
  deliciousFruit1: “Banana”,
  deliciousFruit2: “Apple”, 
  deliciousFruit3: “Orange”
};

Above is an example of the same data shown within our array, which has now been formatted to suit the needs of an object's syntax.

fruits.deliciousFruit1 // Banana
fruits[“deliciousFruit1”] // Banana

Shown above, is the two ways we are able to call our object's data. With the resulting output shown after the two backslashes.

Explain What Functions Are And Why They Are Useful.

What are functions in Javascript?

A function is a set of statements that we use to either execute a specific task or calculate a needed value. The syntax for writing a function starts at the keyword ‘function’ followed by the desired function name, its properties entered into parentheses and the functions statement wrapped in curly brackets. To conduct the intended function, the function must be called/ started by stating the functions name and stating the desired number or string that we wish to run through the said function.

function square(number) {
  return number * number;
}

In the above function named “square”, we have set our function to multiple the value of number by itself. The javascript multiplication operator of ( * ) has been set for this function.

square(5); // 25

Above we have specified the number 5 to be run through our function. The resulting output of our function (5 x 5) is shown following the two backslashes.

Why are Functions Useful?

Functions are used extensively throughout Javascript and are extremely useful. Functions can be executed and re-used multiple times throughout our webpage, all while taking a variety of call inputs. In doing this we save valuable time spent coding multiple lines of the same statement.
